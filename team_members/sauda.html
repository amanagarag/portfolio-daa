Course Learning Reflections

1. What are the kinds of problems we see in nature?

Problems in nature can be broadly categorized into:

  • Iteration: Problems that involve repetitive tasks, such as loops or iterative calculations. Example: Finding the factorial of a number.
  • Recursion: Problems that are solved by dividing them into smaller sub-problems, such as the Tower of Hanoi or Fibonacci sequence.
  • Backtracking: Problems that involve exploring all possible solutions and reverting to previous steps when a solution path fails. Example: Solving mazes or the N-Queens problem.

2. What is space and time efficiency? Why are they important?

Space and time efficiency are critical measures of an algorithm's performance:

  • Time Efficiency: Measures how quickly an algorithm executes. Important to ensure solutions scale well with large inputs.
  • Space Efficiency: Measures the amount of memory an algorithm uses. Essential for resource-constrained environments.

Different classes of problems have varying growth orders, such as:

  • Constant: O(1)
  • Logarithmic: O(log n)
  • Linear: O(n)
  • Quadratic: O(n²)
  • Exponential: O(2ⁿ)

3. Takeaways from Different Design Principles

From Chapter 2, the key design principles include:

  • Divide and Conquer: Break problems into smaller, solvable sub-problems.
  • Dynamic Programming: Solve problems by reusing solutions to overlapping sub-problems.
  • Greedy Algorithms: Choose the optimal choice at each step.
  • Backtracking: Explore all possible solutions systematically.

4. Hierarchical Data and Tree Data Structures

Tree data structures optimize various problem scenarios:

  • Binary Search Trees (BST): Efficient for searching and sorting.
  • AVL Trees: Self-balancing trees ensuring O(log n) operations.
  • Red-Black Trees: Provide balancing with fewer rotations.
  • Heaps: Useful for priority queues and heap sort.
  • Tries: Efficient for storing and retrieving strings, like dictionaries.

5. Array Query Algorithms

Array query algorithms are used to solve problems like range queries and updates. Examples include:

  • Segment Trees: Solve range sum and minimum queries efficiently.
  • Fenwick Trees (BIT): Useful for prefix sums and updates in O(log n).

6. Trees vs. Graphs

Key differences between trees and graphs:

  • Tree: A hierarchical structure with no cycles. Example: File system.
  • Graph: A network of nodes and edges. Can have cycles. Example: Road maps.

Traversals include:

  • Tree: Preorder, Inorder, Postorder
  • Graph: Depth-First Search (DFS), Breadth-First Search (BFS)

7. Sorting and Searching Algorithms

Sorting and searching algorithms connect theory to the real world:

  • Sorting: Bubble sort, quicksort, mergesort optimize ordering problems in databases, logistics, etc.
  • Searching: Binary search enables quick lookups in sorted datasets.

8. Importance of Graph Algorithms

Graph algorithms like Dijkstra’s or Kruskal’s are essential for:

  • Spanning Trees: Minimum cost network connection.
  • Shortest Paths: Optimizing travel or data transfer.

9. Studied Algorithm Design Techniques

Studied techniques include:

  • Divide and Conquer
  • Dynamic Programming
  • Greedy Algorithms
  • Backtracking
