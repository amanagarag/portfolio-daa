1. Kinds of Problems in Nature (Iteration, Recursion, Backtracking)

  • Iteration: Repeatedly perform a set of operations until a condition is met.
    Example: Summing an array.
    T(n) = c * n, where c is a constant.
  • Recursion: Solving a problem by breaking it into smaller subproblems that resemble the original.
    Example: Fibonacci sequence.
    T(n) = T(n-1) + T(n-2) + O(1).
  • Backtracking: Exploring all possible solutions and abandoning paths that donâ€™t lead to solutions.
    Example: N-Queens problem.
    Worst-case T(n) = O(k^n), where k is the branching factor.

2. Space and Time Efficiency

  • Space Efficiency: Measures how much memory an algorithm uses.
    Example: Recursive functions use a call stack.
  • Time Efficiency: Measures how long an algorithm takes to execute.
    Example: Sorting an array takes O(n log n) in merge sort.
  • Importance: Efficient algorithms are critical for scalability and help prevent memory overflows and speed bottlenecks.

Orders of Growth:

Class of Problems Order of Growth Example
Constant O(1) Accessing array element
Logarithmic O(log n) Binary search
Linear O(n) Linear search
Quadratic O(n^2) Bubble sort
Exponential O(2^n) Subset generation

3. Takeaways from Chapter 2 (Design Principles)

  • Divide and Conquer: Split the problem, solve smaller parts, and combine.
  • Greedy Algorithms: Make the locally optimal choice at each step.
  • Dynamic Programming: Solve overlapping subproblems and store results.
  • Backtracking: Abandon partial solutions that fail.

4. Hierarchical Data and Trees

  • Binary Search Tree (BST): O(log n) search/insert/delete for balanced BST.
  • AVL Tree: Self-balancing BST with O(log n) operations.
  • Red-Black Tree: Guarantees balancing with fewer rotations than AVL.
  • Trie: Efficient for prefix-based searches.
  • Heap: Priority queue implementation, O(log n) insert/extract.

Applications:

  • BST: Database indexing.
  • Trie: Autocomplete systems.
  • Heap: Priority scheduling.
  • Red-Black Tree: Associative containers like `std::map`.

5. Array Query Algorithms

  • Need: Efficiently solve range queries and updates.
  • Prefix Sum: Fast range sum queries. Formula: S[j] - S[i-1].
  • Segment Tree: Range queries and updates in O(log n).
  • Fenwick Tree: Efficient for cumulative frequency queries.

6. Tree vs Graph

  • Tree: Hierarchical, acyclic structure. Traversals: Preorder, Inorder, Postorder.
    Applications: Expression evaluation, hierarchical data modeling.
  • Graph: Nodes connected arbitrarily with cycles allowed. Traversals: BFS, DFS.
    Applications: Network routing, dependency resolution.

7. Sorting and Searching Algorithms

Sorting:

  • Bubble Sort: O(n^2), basic sorting.
  • Merge Sort: O(n log n), divide and conquer.
  • Quick Sort: O(n log n) average case, O(n^2) worst case.

Searching:

  • Linear Search: O(n), scans sequentially.
  • Binary Search: O(log n), requires sorted array.
