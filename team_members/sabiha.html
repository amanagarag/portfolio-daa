Understanding Algorithms and Data Structures

1. Types of Problems

  • Iteration: Repeats operations until a condition is met.
    Example: Summing an array. T(n) = c * n
  • Recursion: Solves a problem by reducing it to smaller subproblems.
    Example: Fibonacci sequence. T(n) = T(n-1) + T(n-2) + O(1)
  • Backtracking: Explores all possible solutions, abandoning invalid paths.
    Example: N-Queens problem. T(n) = O(k^n)

2. Space and Time Efficiency

  • Space Efficiency: Measures memory usage.
    Example: Recursive functions use the call stack.
  • Time Efficiency: Measures execution time.
    Example: Sorting an array takes O(n log n) in merge sort.

Orders of Growth

Class of Problems Order of Growth Example
Constant O(1) Accessing an array element
Logarithmic O(log n) Binary search
Linear O(n) Linear search
Quadratic O(n^2) Bubble sort
Exponential O(2^n) Subset generation

3. Design Principles

  • Divide and Conquer: Split problems, solve parts, and combine.
  • Greedy Algorithms: Make locally optimal choices.
  • Dynamic Programming: Solve overlapping subproblems by storing results.
  • Backtracking: Discard unpromising partial solutions.

4. Hierarchical Data and Trees

  • Binary Search Tree (BST): Balanced search/insert/delete in O(log n).
  • AVL Tree: Self-balancing BST.
  • Red-Black Tree: Efficiently balanced tree structure.
  • Trie: Optimized for prefix-based operations.
  • Heap: Priority queue with O(log n) insert/extract.

Applications

  • BST: Database indexing.
  • Trie: Autocomplete systems.
  • Heap: Priority scheduling.
  • Red-Black Tree: Associative containers like `std::map`.

5. Array Query Algorithms

  • Prefix Sum: Efficient range sum queries. Formula: S[j] - S[i-1].
  • Segment Tree: Supports range queries and updates in O(log n).
  • Fenwick Tree: Optimized for cumulative frequency queries.

6. Tree vs Graph

  • Tree: Acyclic hierarchical structure.
    Applications: Expression evaluation, hierarchical data modeling.
  • Graph: Arbitrary connections with cycles allowed.
    Applications: Network routing, dependency resolution.

7. Sorting and Searching Algorithms

Sorting

  • Bubble Sort: O(n^2), basic sorting.
  • Merge Sort: O(n log n), divide and conquer approach.
  • Quick Sort: O(n log n) average case, O(n^2) worst case.

Searching

  • Linear Search: O(n), sequential scan.
  • Binary Search: O(log n), works on sorted data.
